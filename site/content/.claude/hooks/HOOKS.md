# Хуки

Хуки — это пользовательские скрипты, которые запускаются автоматически в определённых точках сессии Claude Code. Они позволяют перехватывать вызовы инструментов, валидировать изменения, обеспечивать соблюдение правил и интегрировать Claude с вашим существующим рабочим процессом разработки.

## Быстрый старт

1. Откройте `.claude/settings.json` (или запустите `/hooks` в сессии)
2. Добавьте запись хука под событием, которое хотите перехватить (например, `PreToolUse`, `PostToolUse`)
3. Напишите скрипт, который читает JSON из stdin, проверяет его и завершается с кодом 0 (разрешить) или 2 (заблокировать)
4. Перезапустите Claude Code. Хуки снимаются снепшотом при запуске

## Как работают хуки

Каждое действие Claude запускает [событие хука](^Именованная точка жизненного цикла, где могут выполняться хуки, например PreToolUse или PostToolUse). Ваша конфигурация указывает Claude Code, какие события отслеживать и что запускать при их срабатывании.

1. **Событие срабатывает**. Claude собирается вызвать инструмент, завершить ответ, начать сессию и т.д.
2. **Matcher проверяет**. Если вы определили matcher (регулярное выражение на имя инструмента или источник события), только совпадающие события запускают ваш хук
3. **Обработчик хука выполняется**. Ваш скрипт (или HTTP-эндпоинт, или LLM-промпт) получает JSON-контекст на stdin
4. **Claude действует по результату**. Код выхода 0 означает разрешить. Код выхода 2 означает заблокировать (для событий, которые это поддерживают). JSON-вывод даёт более тонкий контроль

## События хуков

Хуки срабатывают в этих точках жизненного цикла Claude. События в агентном цикле срабатывают многократно по мере работы Claude:

| Событие | Когда срабатывает | Может блокировать? |
|---|---|---|
| `SessionStart` | Сессия начинается или возобновляется | Нет |
| `UserPromptSubmit` | Вы отправляете промпт, до обработки Claude | Да |
| `PreToolUse` | До выполнения вызова инструмента | Да |
| `PermissionRequest` | Когда должен появиться диалог разрешения | Да |
| `PostToolUse` | После успешного вызова инструмента | Только фидбек |
| `PostToolUseFailure` | После неудачного вызова инструмента | Только фидбек |
| `Stop` | Когда Claude заканчивает ответ | Да (принудительно продолжить) |
| `SubagentStart` | Когда запускается субагент | Нет |
| `SubagentStop` | Когда субагент завершается | Да |
| `Notification` | Когда Claude отправляет уведомление | Нет |
| `TeammateIdle` | Когда член команды агентов собирается бездействовать | Да |
| `TaskCompleted` | Когда задача отмечается как завершённая | Да |
| `ConfigChange` | Когда изменяется файл конфигурации | Да |
| `PreCompact` | До компактификации контекста | Нет |
| `SessionEnd` | Когда сессия завершается | Нет |

## Конфигурация

Хуки определяются в файлах настроек. Три уровня вложенности:

1. **Событие хука**: на какую точку жизненного цикла реагировать
2. **Группа matcher**: регулярное выражение-фильтр (например, только для инструмента `Bash`)
3. **Обработчик хука**: скрипт, эндпоинт, промпт или агент для запуска

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/validate-bash.sh"
          }
        ]
      }
    ]
  }
}
```

### Где определять хуки

| Расположение | Область | Закоммичен? |
|---|---|---|
| `~/.claude/settings.json` | Все ваши проекты | Нет |
| `.claude/settings.json` | Этот проект, общий | Да |
| `.claude/settings.local.json` | Этот проект, персональный | Нет |
| `hooks/hooks.json` плагина | Когда плагин включён | Да |
| Frontmatter навыка/агента | Пока компонент активен | Да |

### Паттерны matcher

`matcher` — это регулярное выражение, сопоставляемое с разными полями в зависимости от события:

- **События инструментов** (`PreToolUse`, `PostToolUse` и т.д.): сопоставляется с `tool_name` (например, `Bash`, `Edit|Write`, `mcp__github__.*`)
- **SessionStart**: сопоставляется с тем, как началась сессия (`startup`, `resume`, `clear`, `compact`)
- **SessionEnd**: сопоставляется с причиной завершения сессии (`clear`, `logout`, `other`)
- **Notification**: сопоставляется с типом уведомления (`permission_prompt`, `idle_prompt`)
- **SubagentStart/Stop**: сопоставляется с типом агента (`Bash`, `Explore`, `Plan`, или пользовательские имена)

Опустите matcher или используйте `"*"` для сопоставления со всем. Некоторые события вроде `UserPromptSubmit` и `Stop` не поддерживают matcher и срабатывают всегда.

### Типы обработчиков хуков

| Тип | Как работает |
|---|---|
| `command` | Запускает shell-команду. Получает JSON на stdin. Общается через коды выхода и stdout |
| `http` | Отправляет JSON как POST-запрос на URL. Тело ответа управляет решением |
| `prompt` | Отправляет ввод модели Claude для однооборотной оценки да/нет |
| `agent` | Запускает субагента с доступом к инструментам (Read, Grep, Glob) для проверки условий |

Не все события поддерживают все типы. `SessionStart`, `Notification`, `PreCompact` и другие поддерживают только хуки типа `command`.

## Ввод и вывод

### Что получает ваш хук

Все хуки получают этот JSON на stdin (или как тело POST для HTTP-хуков):

```json
{
  "session_id": "abc123",
  "cwd": "/home/user/my-project",
  "permission_mode": "default",
  "hook_event_name": "PreToolUse",
  "tool_name": "Bash",
  "tool_input": {
    "command": "npm test"
  }
}
```

Поля `tool_name` и `tool_input` специфичны для события. Каждое событие добавляет свои поля поверх общих.

### Коды выхода

| Код | Значение |
|---|---|
| `0` | Успех. Claude Code парсит stdout для опционального JSON-вывода |
| `2` | Блокирующая ошибка. stderr передаётся Claude как сообщение об ошибке. Эффект зависит от события |
| Другой | Неблокирующая ошибка. stderr показывается в verbose-режиме, выполнение продолжается |

### JSON-вывод

Для более тонкого контроля завершитесь с кодом 0 и выведите JSON в stdout вместо использования только кодов выхода:

```json
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "deny",
    "permissionDecisionReason": "Деструктивная команда заблокирована"
  }
}
```

Универсальные JSON-поля работают во всех событиях:

| Поле | Эффект |
|---|---|
| `continue` | Если `false`, Claude полностью прекращает обработку |
| `stopReason` | Сообщение, показываемое пользователю при `continue` = false |
| `suppressOutput` | Если `true`, скрывает stdout из verbose-режима |
| `systemMessage` | Предупреждающее сообщение, показываемое пользователю |

## Асинхронные хуки

Добавьте `"async": true` к command-хуку для запуска в фоне. Claude продолжает работу немедленно, пока хук выполняется. Результаты доставляются на следующем шаге разговора.

```json
{
  "type": "command",
  "command": ".claude/hooks/run-tests.sh",
  "async": true,
  "timeout": 120
}
```

Асинхронные хуки не могут блокировать вызовы инструментов или возвращать решения. Они полезны для запуска наборов тестов, деплоев или уведомлений параллельно с работой Claude.

## Типичные паттерны

**Блокировка опасных команд** (PreToolUse на Bash): парсите `tool_input.command`, проверяйте на `rm -rf` или другие деструктивные паттерны, завершайтесь с кодом 2 для блокировки.

**Автоформатирование после редактирования** (PostToolUse на Write|Edit): запускайте линтер или форматтер на изменённом файле, передайте вывод обратно Claude.

**Проверка типов после изменений** (PostToolUse на Write|Edit): запускайте `tsc --no-emit` после изменений TypeScript, Claude видит ошибки и исправляет места вызова.

**Предотвращение дублирования кода** (PostToolUse на Write): запустите вторичный экземпляр Claude для ревью нового кода по сравнению с существующими паттернами.

**Проверки качества перед остановкой** (Stop): проверьте, что тесты проходят или требуемые файлы существуют, прежде чем позволить Claude завершиться.

**Инъекция окружения при запуске** (SessionStart): запишите выражения `export` в `$CLAUDE_ENV_FILE` для установки переменных окружения для сессии.

## Советы

- Хуки снимаются снепшотом при запуске сессии. Перезапустите Claude после изменения конфигурации хуков
- Используйте `$CLAUDE_PROJECT_DIR` в командах для ссылки на скрипты относительно корня проекта
- Используйте `/hooks` в сессии для просмотра, добавления и удаления хуков интерактивно
- Запустите `claude --debug`, чтобы увидеть детали выполнения хуков и коды выхода
- Установите `"disableAllHooks": true` в настройках для временного отключения всех хуков
- Держите хуки быстрыми. Таймаут по умолчанию — 600 секунд для команд, 30 для промптов
- Цитируйте shell-переменные (`"$VAR"` а не `$VAR`) и валидируйте входные данные. Скрипты хуков работают с вашими полными правами пользователя
- Для MCP-инструментов используйте паттерны вроде `mcp__github__.*` или `mcp__.*__write.*`
- Хуки prompt и agent возвращают `{"ok": true/false, "reason": "..."}` вместо использования кодов выхода
