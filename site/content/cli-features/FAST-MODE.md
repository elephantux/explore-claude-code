# Fast Mode

Fast mode — высокоскоростная конфигурация для Claude Opus 4.6, делающая модель в 2.5 раза быстрее при более высокой стоимости за токен. Включайте его командой `/fast` когда нужна скорость для интерактивной работы.

## Быстрый старт

1. Введите `/fast` и нажмите Tab для переключения
2. Появится подтверждение: «Fast mode ON»
3. Иконка `↯` появится рядом с промптом пока fast mode активен
4. `/fast` снова — для выключения

Fast mode — это **не другая модель**. Используется тот же Opus 4.6 с другой конфигурацией API, приоритизирующей скорость над стоимостью.

## Стоимость

| Режим | Input (MTok) | Output (MTok) |
|---|---|---|
| Fast mode на Opus 4.6 (<200K) | $30 | $150 |
| Fast mode на Opus 4.6 (>200K) | $60 | $225 |

При включении fast mode посреди разговора вы платите полную цену за некэшированные входные токены за весь контекст. Выгоднее включать с самого начала сессии.

## Когда использовать

**Fast mode подходит для:**
- Быстрой итерации над изменениями кода
- Живых сессий отладки
- Срочной работы с жёсткими дедлайнами

**Стандартный режим лучше для:**
- Длинных автономных задач
- Пакетной обработки или CI/CD
- Чувствительных к стоимости нагрузок

## Fast mode vs Effort level

| Настройка | Эффект |
|---|---|
| **Fast mode** | То же качество модели, меньше задержка, выше стоимость |
| **Lower effort** | Меньше времени на размышления, быстрее ответы, потенциально ниже качество |

Можно комбинировать оба: fast mode с низким [effort level](^Настраивается через /model или конфиг. Влияет на глубину размышлений Claude) для максимальной скорости на простых задачах.

## Требования

- **Не доступен** на сторонних облачных провайдерах (Amazon Bedrock, Google Vertex AI, Microsoft Foundry)
- Нужен включённый **extra usage** — биллинг сверх включённого в план
- Для **Teams и Enterprise** — администратор должен явно включить fast mode

## Персессионная активация

По умолчанию fast mode сохраняется между сессиями. Администраторы могут требовать активацию в каждой сессии:

```json
{
  "fastModePerSessionOptIn": true
}
```

При этой настройке каждая сессия начинается с выключенным fast mode.

## Rate limits

Fast mode имеет отдельные rate limits от стандартного Opus 4.6. При достижении лимита:

1. Fast mode автоматически откатывается на стандартный Opus 4.6
2. Иконка `↯` становится серой (cooldown)
3. Вы продолжаете работать на стандартной скорости и ценах
4. Когда cooldown истекает, fast mode автоматически возвращается

## Настройка через settings.json

```json
{
  "fastMode": true
}
```

Или отключите полностью через переменную окружения: `CLAUDE_CODE_DISABLE_FAST_MODE=1`
