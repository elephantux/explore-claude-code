# Checkpointing

Claude Code автоматически отслеживает редактирования файлов по мере работы, позволяя быстро отменять изменения и откатываться к предыдущим состояниям.

## Как это работает

1. **Автоматическое отслеживание**: Каждый пользовательский промпт создаёт новый checkpoint
2. **Персистентность**: Checkpoints сохраняются между сессиями и доступны в возобновлённых разговорах
3. **Автоочистка**: Удаляются вместе с сессиями через 30 дней (настраивается)

## Rewind & Summarize

Нажмите `Esc` дважды (`Esc` + `Esc`) или используйте `/rewind` для открытия меню отката. Прокручиваемый список показывает каждый ваш промпт из сессии.

### Доступные действия

| Действие | Что делает |
|---|---|
| **Restore code and conversation** | Откатить и код, и разговор к этой точке |
| **Restore conversation** | Откатить к этому сообщению, сохранив текущий код |
| **Restore code** | Откатить изменения файлов, сохранив разговор |
| **Summarize from here** | Сжать разговор с этой точки в саммари |
| **Never mind** | Вернуться к списку сообщений |

После восстановления или суммаризации оригинальный промпт из выбранного сообщения восстанавливается в поле ввода.

## Restore vs Summarize

Три опции restore откатывают состояние: отменяют изменения кода, историю разговора или и то, и другое.

**Summarize from here** работает иначе:
- Сообщения до выбранного остаются нетронутыми
- Выбранное сообщение и все последующие заменяются компактным AI-саммари
- Файлы на диске не меняются
- Оригинальные сообщения сохраняются в транскрипте сессии

Это похоже на `/compact`, но таргетированно: вы сохраняете ранний контекст полностью и сжимаете только части, занимающие место.

## Типичные сценарии

- **Исследование альтернатив**: Пробуйте разные подходы к реализации без потери стартовой точки
- **Восстановление после ошибок**: Быстро отмените изменения, внёсшие баги
- **Итерация над фичами**: Экспериментируйте с вариациями, зная что можете откатиться
- **Освобождение контекста**: Суммаризуйте многословную сессию отладки, сохранив начальные инструкции

## Ограничения

### Bash-команды не отслеживаются

Checkpointing не отслеживает файлы, изменённые bash-командами:

```bash
rm file.txt
mv old.txt new.txt
cp source.txt dest.txt
```

Эти изменения файлов нельзя отменить через rewind. Отслеживаются только прямые редактирования через инструменты Claude.

### Внешние изменения не отслеживаются

Checkpointing отслеживает только файлы, отредактированные в текущей сессии. Ручные изменения, сделанные вами вне Claude Code, и правки из других параллельных сессий обычно не захватываются.

### Не замена системе контроля версий

Checkpoints предназначены для быстрого восстановления на уровне сессии:

- Продолжайте использовать Git для коммитов, веток и долгосрочной истории
- Checkpoints дополняют, но не заменяют полноценный контроль версий
- Думайте о checkpoints как о «локальном undo», а о Git как о «постоянной истории»
