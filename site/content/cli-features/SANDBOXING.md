# Sandboxing

Claude Code использует нативную песочницу для обеспечения более безопасной среды выполнения агента. Вместо запроса разрешения на каждую bash-команду, песочница создаёт определённые границы, в которых Claude может работать свободнее с меньшим риском.

## Почему это важно

Традиционная безопасность на основе разрешений требует постоянных подтверждений. Это ведёт к:

- **Усталости от одобрений**: Постоянные клики «approve» снижают внимание
- **Снижению продуктивности**: Постоянные прерывания замедляют работу
- **Ограниченной автономии**: Claude не может эффективно работать, ожидая одобрений

Песочница решает эти проблемы:

1. **Чёткие границы**: Точно какие директории и хосты доступны
2. **Меньше запросов**: Безопасные команды внутри песочницы не требуют одобрения
3. **Безопасность**: Попытки доступа за пределы песочницы вызывают уведомления
4. **Автономность**: Claude может работать более независимо в рамках лимитов

## Как это работает

### Изоляция файловой системы

- **Запись по умолчанию**: Чтение и запись в текущую рабочую директорию и её поддиректории
- **Чтение по умолчанию**: Чтение всего компьютера, кроме определённых запрещённых директорий
- **Блокировка**: Нельзя изменять файлы вне рабочей директории без явного разрешения
- **Настраиваемость**: Определите свои разрешённые и запрещённые пути

### Сетевая изоляция

- **Ограничения доменов**: Только одобренные домены доступны
- **Подтверждение пользователя**: Запросы к новым доменам вызывают диалоги разрешений
- **Пользовательский прокси**: Продвинутые пользователи могут реализовать свои правила
- **Полное покрытие**: Ограничения применяются ко всем скриптам и subprocess'ам

### OS-уровень

- **macOS**: Использует Seatbelt для песочницы
- **Linux**: Использует [bubblewrap](https://github.com/containers/bubblewrap) для изоляции
- **WSL2**: Использует bubblewrap, как Linux
- **WSL1**: Не поддерживается (bubblewrap требует kernel-фичи WSL2)

## Включение

Запустите команду `/sandbox`:

```
/sandbox
```

Откроется меню для выбора режима песочницы.

### Режимы

| Режим | Поведение |
|---|---|
| **Auto-allow** | Команды в песочнице выполняются автоматически без запроса. Команды, которые нельзя песочничить, проходят обычный flow разрешений |
| **Regular permissions** | Все команды проходят стандартный flow разрешений, даже в песочнице |

## Конфигурация

### Разрешить запись в дополнительные пути

```json
{
  "sandbox": {
    "enabled": true,
    "filesystem": {
      "allowWrite": ["~/.kube", "//tmp/build"]
    }
  }
}
```

### Префиксы путей

| Префикс | Значение | Пример |
|---|---|---|
| `//` | Абсолютный путь от корня | `//tmp/build` → `/tmp/build` |
| `~/` | Относительно home | `~/.kube` → `$HOME/.kube` |
| `/` | Относительно директории settings-файла | `/build` → `$SETTINGS_DIR/build` |
| `./` или без | Относительный путь | `./output` |

### Сетевые настройки

```json
{
  "sandbox": {
    "network": {
      "allowedDomains": ["github.com", "*.npmjs.org"],
      "allowUnixSockets": ["/var/run/docker.sock"],
      "allowLocalBinding": true
    }
  }
}
```

### Исключение команд

Некоторые команды несовместимы с песочницей:

```json
{
  "sandbox": {
    "excludedCommands": ["docker", "watchman"]
  }
}
```

## Защита

### От prompt injection

Даже если атакующий успешно манипулирует поведением Claude, песочница обеспечивает безопасность:

**Файловая система:**
- Нельзя изменить критические конфиги вроде `~/.bashrc`
- Нельзя изменить системные файлы в `/bin/`
- Нельзя читать файлы, запрещённые в настройках разрешений

**Сеть:**
- Нельзя эксфильтровать данные на серверы атакующего
- Нельзя загружать вредоносные скрипты с неавторизованных доменов
- Нельзя делать неожиданные API-вызовы

## Ограничения

- **Сетевая фильтрация**: Система фильтрует домены, но не инспектирует трафик. Разрешайте только доверенные домены
- **Unix-сокеты**: `allowUnixSockets` может дать доступ к мощным сервисам (например, `/var/run/docker.sock`)
- **Широкие разрешения записи**: Слишком широкие права могут привести к эскалации привилегий
- **Производительность**: Минимальные накладные расходы, но некоторые файловые операции могут быть немного медленнее
